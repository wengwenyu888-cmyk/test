## 🛠️ Git 核心概念

在正式敲命令之前，理解 Git 的**四个核心区域**至关重要。

1. **工作区 (Workspace)**：你电脑上实际看到并正在修改的目录（正在写代码的地方）。
2. **暂存区 (Staging Area / Index)**：一个临时准备提交的区域。就像打包快递，先把要寄的东西放进箱子里。
3. **本地仓库 (Local Repository)**：安全存放版本历史的地方（在本地 `.git` 文件夹中）。一旦提交到这里，你的代码就有了“历史存盘”。
4. **远程仓库 (Remote Repository)**：托管在网络上的服务器（如 GitHub、Gitee、GitLab），用于团队协作和代码备份。

## 🚀 基础通关：最常用的 5 步常规工作流

绝大多数情况下，你每天的 Git 操作只会重复以下 5 步：

### 1. 初始化或克隆

- **自己新建项目**：在项目根目录下执行，将其变成 Git 仓库。

  ```bash
  git init
  ```

- **下载别人的项目**：直接把远程仓库克隆到本地。

  ```bash
  git clone <远程仓库地址>
  ```

### 2. 查看当前状态

随时随地都可以输入这行命令，查看哪些文件被修改了、哪些还没准备提交。**它是你的“指路明灯”。**

```bash
git status
```

### 3. 把修改添加到暂存区（打包）

- 添加指定文件：

  ```bash
  git add main.py
  ```

- 添加当前目录下的**所有**修改（最常用）：

  ```bash
  git add .
  ```

### 4. 提交到本地仓库（存档）

把暂存区的内容正式保存下来，必须附带一条有意义的提交信息，解释你改了什么。

```bash
git commit -m "feat: 新增用户登录功能"
```

### 5. 推送到远程仓库（同步）

把本地的提交同步到远程服务器（假设分支名为 `main`）。

```bash
git push origin main
```

> 💡 **小提示**：如果别人也改了代码，你在推送前需要先用 `git pull` 把远程的最新的代码“拉”下来融合一下，免得发生冲突。

## 🌿 分支管理：开启平行宇宙

分支（Branch）是 Git 最强大的功能。它允许你从主线中切出一个“平行宇宙”去开发新功能，就算代码写砸了，也不会影响到主线（`main` 或 `master`）的正常运行。

- **查看所有分支**（带 `*` 的是当前所在分支）：

  ```bash
  git branch
  ```

- **新建并切换到新分支**（比如开发社交功能）：

  ```bash
  git switch -c feature-social
  ```

- **切回主分支**：

  ```bash
  git switch main
  ```

- **合并分支**（把 `feature-social` 的成果合并到当前所在的 `main` 分支）：

  ```bash
  git merge feature-social
  ```

## 📝 Git 常用命令极简速查表

| **命令**              | **作用**             | **对应区域变化**     |
| --------------------- | -------------------- | -------------------- |
| `git init`            | 初始化本地仓库       | 创建本地仓库         |
| `git status`          | 查看当前文件状态     | 诊断工具             |
| `git add <文件>`      | 将文件标记为准备提交 | 工作区 -> 暂存区     |
| `git commit -m "..."` | 记录一次历史快照     | 暂存区 -> 本地仓库   |
| `git log`             | 查看历史提交记录     | 浏览本地仓库历史     |
| `git push`            | 推送本地修改到远程   | 本地仓库 -> 远程仓库 |
| `git pull`            | 拉取远程修改并合并   | 远程仓库 -> 工作区   |
